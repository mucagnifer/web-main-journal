# web-main-journal

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容不针对任何特定域名或网站，仅作为通用 HTML 文档的存储与展示空间。仓库结构简单，适合存放静态页面、演示文稿、学习笔记或临时页面等。

## 目录说明

```
web-main-journal/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放独立 HTML 页面
│   ├── example1.html
│   └── example2.html
└── assets/             # 公共资源文件（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：每个 HTML 文件对应一个独立页面，互不关联。
- `assets/`：存放页面引用的公共资源，便于统一管理。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接在浏览器中打开运行。
- 页面之间不共享数据或状态，仅通过静态资源（如 CSS、JS）实现统一风格。
- 文件名应具有描述性，便于识别内容（例如 `notes.html`、`demo.html`）。
- 不限制页面主题、用途或技术栈（支持纯 HTML、CSS、JavaScript 或嵌入式框架）。

## 维护说明

- 本仓库仅用于归档和发布，不涉及数据库、后端服务或用户交互。
- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并更新 `index.html` 中的链接（如有）。
- 修改公共资源时，注意保持与现有页面的兼容性。
- 建议定期检查页面链接和资源引用，避免失效或错误。
- 本仓库不接收外部 PR 或 issue，仅由仓库维护者自主管理。
